# pm-execution

执行和产品管理技能：PRD、OKR、路线图、Sprint、事前分析 (Pre-mortems)、利益相关者地图、用户故事、优先级框架等。

## 技能 (15)

- **brainstorm-okrs** — 集思广益，制定与公司目标一致的团队级 OKR。
- **create-prd** — 使用包含摘要、背景、目标、细分市场、价值主张、解决方案详情和发布计划在内的 8 个部分的完整模板创建产品需求文档。
- **dummy-dataset** — 生成用于测试的真实虚拟数据集，具有可定制的列、约束和输出格式（CSV、JSON、SQL、Python 脚本）。
- **job-stories** — 使用“当 [情景] 时，我想要 [动机]，以便我能够 [结果]”格式创建具有详细验收标准的 Job Stories。
- **outcome-roadmap** — 将以产出 (Output) 为中心的路线图转化为以成果 (Outcome) 为中心的路线图。
- **pre-mortem** — 对 PRD 进行事前分析 (Pre-mortem)。
- **prioritization-frameworks** — 包含 9 个优先级排序框架的参考指南，附带公式、适用场景指导和模板。
- **release-notes** — 根据工单、PRD 或变更日志生成面向用户的发布说明。
- **retro** — 协助组织结构化的 Sprint 回顾会议。
- **sprint-plan** — 规划 Sprint，包括容量估算、故事选择、依赖映射和风险识别。
- **stakeholder-map** — 使用权力/利益矩阵构建利益相关者地图，识别每个象限的沟通策略，并生成沟通计划。
- **summarize-meeting** — 将会议记录总结为结构化模板，包含日期、参与者、主题、摘要要点和行动项。
- **test-scenarios** — 从用户故事中创建全面的测试场景，包含测试目标、启动条件、用户角色、分步操作和预期结果。
- **user-stories** — 遵循 3C（卡片、对话、确认）和 INVEST 标准创建用户故事，包含描述、设计链接和验收标准。
- **wwas** — 以 Why-What-Acceptance 格式创建产品待办事项。

## 命令 (10)

- `/pm-execution:generate-data` — 生成用于测试的真实虚拟数据集 —— CSV、JSON、SQL 插入或 Python 脚本。
- `/pm-execution:meeting-notes` — 将会议记录总结为结构化笔记，包含决策、行动项和后续工作。
- `/pm-execution:plan-okrs` — 集思广益制定与公司目标一致的团队级 OKR —— 具有可衡量关键结果的定性目标。
- `/pm-execution:pre-mortem` — 对 PRD、发布计划或功能进行事前风险分析 —— 在问题发生之前识别潜在风险。
- `/pm-execution:sprint` — Sprint 生命周期 —— 规划 Sprint、进行回顾或生成发布说明。
- `/pm-execution:stakeholder-map` — 在权力 × 利益矩阵上映射利益相关者，并创建量身定制的沟通计划。
- `/pm-execution:test-scenarios` — 从用户故事或功能规格中生成全面的测试场景 —— 正常路径、边缘案例和错误处理。
- `/pm-execution:transform-roadmap` — 将基于功能的路线图转换为传达战略意图的以成果为中心的路线图。
- `/pm-execution:write-prd` — 从功能构思或问题陈述中创建完整的产品需求文档。
- `/pm-execution:write-stories` — 将功能分解为待办事项 —— 用户故事、Job Stories 或带有验收标准的 WWA 格式。
